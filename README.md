# 👋 Jay / Deokjae Kwon

#### 성능을 높이면서도 안정성을 놓치지 않는 백엔드 개발자입니다.

트래픽이 집중되는 프로젝트 랭킹 API에 Redis 캐싱 전략을 적용해 DB 부하를 완화하고,  
Redisson 분산 락으로 스냅샷 캐시의 중복 생성을 방지해 동시성 이슈를 해결했습니다.  
또한 캐시 장애 시 DB로 자동 전환되는 fallback 이중 방어 로직을 도입해 안정성을 강화했습니다.


#### 검증과 개선을 통해 시스템의 신뢰성을 높입니다.

동시성 테스트 코드를 작성해 멀티스레드 환경에서 스냅샷 생성의 단일 실행 여부를 검증했으며,  
k6 부하 테스트 결과 p99 응답 시간이 약 32.3%, 평균 응답 시간이 41.5% 개선되었습니다.
